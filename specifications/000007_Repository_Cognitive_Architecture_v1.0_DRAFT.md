# STRAUSSIAN DOCUMENTARY MEMORY
# REPOSITORY COGNITIVE ARCHITECTURE

Version: 1.0
Document Number: 000007
Classification: Foundational Repository Specification
Status: Draft for Cross-Document Constitutional Review

Authority: Establishes repository-level architectural principles for memory, cognition, certification, and graph projection without replacing the existing foundational specifications.

Purpose

This draft establishes the repository architecture by which recovered documentary practice becomes durable inquiry memory, reusable cognitive operation, and certified repository cognition. It does not certify any repository object, redefine existing ontology classes, or replace the authority of the existing specifications. It is a draft for cross-document review and later compatibility audit.

I. REPOSITORY MISSION

The Repository exists to recover, certify, accumulate, and apply the documentary operations by which Leo Strauss conducted inquiry, so that future LLM investigations increasingly proceed through recovered Straussian documentary discipline rather than ad hoc interpretation.

II. THREE-LEVEL AUTHORITY ARCHITECTURE

The repository architecture is organized in three levels.

1. Constitutional architecture
   - Defines what must remain true.
   - Establishes the enduring commitments of repository identity, validity, authority, and preservation.
   - Determines the boundaries that may not be crossed by implementation convenience.

2. Normative specifications
   - Define stable repository contracts.
   - Govern ontology, identifiers, relationship assertions, vocabulary, admission, lifecycle, and documentary governance.
   - Provide the formal rules that implementations must preserve.

3. Implementation
   - Includes GitHub, Neo4j, schemas, validators, workflows, directories, and software.
   - Provides operational means for preserving and projecting repository cognition.
   - May evolve without becoming constitutional rules merely because a chosen implementation uses them.

Implementation details must not become constitutional rules. The architecture may be implemented through different tools, file structures, or software systems so long as the constitutional and normative commitments remain intact.

III. REPOSITORY MEMORY ARCHITECTURE

The repository memory architecture is divided into three domains.

A. Documentary Memory

Documentary Memory preserves the documentary world of inquiry:
- Works;
- Witnesses;
- Passages; and
- their documentary relations.

Documentary Memory is the domain of recoverable documentary structure, evidence, and relation.

B. Inquiry Memory

Inquiry Memory preserves the record of inquiry itself:
- preserved investigations;
- evidence paths;
- uncertainty;
- reconstructions;
- comparisons;
- decisions;
- corrections; and
- certification history.

Inquiry Memory preserves the history of how the Repository came to its present understanding.

C. Cognitive Memory

Cognitive Memory preserves certified reusable documentary operations recovered from completed inquiries.

Cognitive Memory is not the same as raw inquiry history. It is the repository’s durable stock of validated acts of investigation that may be reused in later inquiry.

IV. COGNITIVE ACQUISITION

A completed inquiry does not automatically enlarge Cognitive Memory.

A reusable operation must pass through the following sequence:

Certified Inquiry → Candidate Cognitive Operation → Validation → Certification → Cognitive Acquisition → Integration

The sequence is normative. It is not sufficient that an inquiry has concluded, or that a procedure appears useful, for the operation to become cognitive memory.

A candidate operation may be preserved before certification, but it may not govern future inquiry until it has been integrated into the repository’s cognitive architecture.

The distinction between preserved candidate operation and integrated cognitive memory is constitutional for future inquiry governance.

V. DOCUMENTARY OPERATION DISTINCTION

This draft distinguishes between two related but distinct categories.

A. Documentary events represented in the existing ontology

The existing ontology already recognizes documentary events such as:
- citation;
- quotation;
- omission;
- placement; and
- sequence.

These are documentary events represented in the documentary ontology and belong to the record of documentary structure and use.

B. Cognitive documentary operations

Cognitive documentary operations are reusable investigative acts recovered from Strauss’s practice. They are not merely named events in the documentary record. They are operations that have been recovered, validated, certified, and made reusable for future inquiry.

Terminology Conflict

This draft explicitly records a terminology conflict requiring later controlled resolution:
- The existing ontology uses the term “Documentary Operation” for documentary events such as citation, quotation, omission, placement, and sequence.
- This architecture uses “cognitive documentary operation” for reusable investigative acts recovered from Strauss’s practice.

This draft does not silently redefine the existing ontology term. The conflict must be resolved later through a controlled cross-document amendment.

VI. CANONICAL RECORD AND GRAPH PROJECTION

The full certified documentary artifact is authoritative over every graph, index, database, retrieval system, or API projection.

For the present implementation:
- GitHub is the canonical versioned documentary record.
- Neo4j is a derived cognitive graph projection.
- Neo4j internal identifiers are non-authoritative.
- Neo4j must be reproducibly rebuildable from canonical repository artifacts.
- No direct Neo4j modification may silently alter canonical Repository cognition.

The graph is therefore derivative. It is a projection and instrument of retrieval and traversal, not an independent source of doctrine or authority.

VII. GRAPH REPRESENTATION PRINCIPLE

Constitutionally significant relationships must preserve:
- stable identity;
- predicate;
- subject;
- object;
- documentary basis;
- provenance;
- lifecycle;
- certification; and
- revision history.

Neo4j direct edges may be generated for traversal performance, but they must not replace authoritative Relationship Assertion records. Graph edges may accelerate use and retrieval, but they are not a substitute for the canonical relational record.

VIII. NON-DESTRUCTIVE AMENDMENT AND CROSS-DOCUMENT RECONCILIATION

This draft does not declare the existing ontology invalid. It does not replace or supersede the foundational specifications. It identifies the need for a future cross-document amendment to reconcile the following domains:

- Hermeneutic Objects;
- Comparative Reconstructions;
- Inquiry entities;
- Documentary Operations;
- Cognitive Acquisition Objects;
- Cognitive Memory; and
- the Neo4j graph projection.

That amendment must preserve the existing authoritative specifications while resolving the conceptual overlaps and boundaries between inquiry, interpretation, cognitive acquisition, and graph projection.

IX. STATUS, SCOPE, AND REVIEW CONTROL

This draft is limited to this specification file. It establishes architecture and review obligations only. It does not certify any object, create new constitutional entities, or alter any existing constitutional or repository file.

This draft is intended for cross-document constitutional review. Certification requires a later focused compatibility audit.

X. FINAL DECLARATION

This draft establishes a repository architecture in which documentary memory, inquiry memory, and cognitive memory are distinct but interdependent. It preserves the distinction between documentary evidence and reusable cognitive operation. It requires that the canonical documentary artifact remain authoritative over all projections, including graph and retrieval systems. It preserves unresolved cross-document conflicts for later controlled amendment rather than silently rewriting existing repository law.
