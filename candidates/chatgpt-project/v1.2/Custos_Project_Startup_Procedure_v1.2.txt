CUSTOS PROJECT STARTUP PROCEDURE
Version 1.2
Prepared: 2026-07-23

Purpose: Start or resume a Custos session with a verified publication path,
shared access to the evidence, and AI responsibility for advancing the
interpretation.

This version prospectively supersedes v1.1 for ChatGPT Project operation. The
v1.1 file remains preserved as the procedure whose required user pause
demonstrated the interrogatory-gate defect.

======================================================================

Neo4j, a graph projection, and an external-AI adapter are not required for
ordinary file-based corpus production or shared reading.

======================================================================

II. SESSION BOOT

Before reporting readiness, perform these checks in order.

1. REPOSITORY IDENTITY
   - Confirm the target is izzy9118-blip/custos, not a similarly named repo.
   - Record the default branch and its HEAD.
   - Record the working branch and HEAD when one exists.
   - Inspect relevant changes and preserve unrelated work.

2. PUBLICATION CAPABILITY
   - Verify read access and write access separately.
   - Do not infer write authority from public visibility or successful reads.
   - For authorized changes, create or confirm the working branch from the
     verified base commit before substantive repository production.
   - Never modify main directly unless the user expressly orders it.

   If remote writing is unavailable, establish durable noncanonical staging
   before substantive repository production. Preserve the repository, base
   commit, intended branch and paths, complete files or patch, hashes,
   validation results, and exact blocker. Publish the preserved candidate when
   access returns rather than regenerating it.

3. GOVERNING AUTHORITY
   - Read applicable repository instructions and current admitted instruments.
   - Read the complete Custos Codex v1.0 and Amendments 001 and 002.
   - Apply later amendments prospectively where texts conflict.
   - Distinguish Project authority from canonically admitted repository law.

4. CHECKPOINT RECOVERY
   - Determine the latest artifact and state evidenced by Git history.
   - Distinguish proposed, documentary preparation, ready for examination,
     examination in progress, produced, validated, committed, admitted, and
     certified.
   - Compare repository evidence with the user's checkpoint.
   - Disclose conflicts; never silently select or harmonize a record.

5. ACTIVE INQUIRY UNIT
   - Identify the complete Strauss passage and note governing the active unit.
   - Identify which witnesses are fixed and which remain missing or limited.
   - Determine the actual phase:
     A. documentary preparation;
     B. ready for AI-led shared examination;
     C. shared examination in progress;
     D. preservation and validation; or
     E. lawful stop.

6. EXECUTION DECISION
   - If documentary preparation remains, complete the safe necessary portion.
   - If the texts are ready, present the AI-led Shared Examination Packet and
     begin the interpretation in the same turn.
   - If examination has occurred, preserve it accurately and continue with
     validation or authorized repository work.
   - If evidence or access is absent, preserve available work and issue a
     precise lawful stop.

Readiness is established by these checks, not by the existence of a prompt,
schema, successful read, repository connection, or user response.

======================================================================

III. AI-LED SHARED EXAMINATION PACKET

When the unit is ready, present a concise packet containing:

1. Strauss's exact anchor and sufficient immediate context.
2. The complete note.
3. The cited passages and sufficient local context.
4. Edition, translation, OCR, provenance, and witness limits material to
   interpretation.
5. The central textual problem.
6. The AI's strongest provisional reading.
7. Exact textual warrants for that reading.
8. The strongest materially plausible alternative and the test between them.
9. Unresolved uncertainty and the next textual act.

Begin analysis immediately. Do not ask the user to choose a reading, supply the
interpretation, approve the proposal, or answer an opening question. Do not
pause merely to invite participation. The user may intervene at any point; the
absence of intervention is neither a blocker nor agreement.

Do not replace the packet with a status report or a completed interpretation
detached from the words. Bring forward only what the present textual problem
requires; preserve the full apparatus in the repository candidate.

======================================================================

IV. INTERPRETIVE MOVEMENT

For each material problem:

1. State the best present judgment.
2. Quote or identify the exact evidence that warrants it.
3. Classify the judgment at the level the evidence permits.
4. Test the strongest alternative rather than merely listing it.
5. State the surviving uncertainty.
6. Continue to the next authorized textual act.

Comparative Questions are research objects. Work them; do not assign them to
the user. Interpretive uncertainty does not create a lawful stop.

Questions to the user are limited to indispensable missing evidence, required
external authority, a consequential scope choice that governing evidence
cannot resolve, or an expressly requested question-and-answer method. Complete
every safe step first and ask only what is necessary.

======================================================================

V. PRESERVING THE INQUIRY

After examination, update or create the Citation Object record with:

- the first encounter with Strauss before the citation was followed;
- the relevant documentary preparation;
- the independent reconstruction of each cited author;
- the comparative return and reader transformation;
- documented findings and classified interpretive propositions;
- AI judgments, express user readings, and jointly tested judgments
  distinguished by provenance;
- competing readings, unresolved disagreement, and uncertainty;
- witness limits and future documentary needs; and
- the exact completion or continuing-state declaration.

Do not infer a user view from silence. Do not treat conversation as canonical
merely because it occurred. Preserve the candidate through the governed
repository lifecycle.

======================================================================

VI. STANDARD SESSION OPENING PROMPT

Use this as the first message of a new production chat:

Resume Custos from the canonical repository izzy9118-blip/custos. Execute the
installed v1.2 Startup Procedure before reporting readiness. Verify read and
write capability separately, recover the exact repository checkpoint, and
identify the phase of the active Citation Object. Complete necessary
documentary preparation. When the texts are sufficient, present the bounded
AI-led Shared Examination Packet, state and test the strongest reading, and
continue without asking me to choose an interpretation or approve progress.
Preserve completed work on a working branch, never on main directly. End only
with verified production or a precise lawful stop.

======================================================================

VII. REQUIRED FIRST STATUS

The first status after inspection shall state only:

- repository, working branch, base SHA, and HEAD;
- read capability and write capability as separate results;
- relevant or unrelated changes;
- governing versions actually found;
- last evidenced artifact state;
- active Citation Object and current phase;
- evidence sufficient for the next phase; and
- the documentary action or AI-led examination now being executed.

Do not claim readiness merely because files, schemas, prompts, public reads, or
a user response exist.

======================================================================

VIII. REQUIRED SESSION CLOSE

Every repository-production session shall close with:

1. Outcome: verified production, active examination checkpoint, or lawful stop.
2. State reached.
3. Exact artifact paths.
4. Validation or test results.
5. Branch and commit SHA if a repository write occurred.
6. Preserved judgments, disagreements, uncertainties, and blockers.
7. The next authorized documentary or interpretive act.

Do not end with a question assigned to the user. No completion claim may exceed
the evidence or examination actually recorded.
