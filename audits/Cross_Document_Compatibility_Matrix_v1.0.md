## EFFECT ON SPECIFICATIONS

### Specification 000001 — Documentary Ontology

No class amendment is presently required.

Clarification is achieved by mapping cognition to existing classes and by preserving Documentary Operation for documentary acts and events.

### Specification 000002 — Identifier Specification

No new identifier family is presently required.

Existing families provide:

- IAR for procedures;
- INT and SPT for literary components;
- HO for Taxonomy;
- VAL for validation;
- CER for certification;
- DDR for integration decisions;
- REG for registers;
- MAN for manifests.

### Specification 000003 — Relationship Specification

One focused amendment is required for inferred relationship representation.

### Specification 000004 — Controlled Vocabulary Specification

No immediate amendment is required under the register-and-decision model.

DOCUMENTED FINDING and SUPPORTED INFERENCE retain their existing EPC meanings.

### Specification 000005 — Constitutional Admission Protocol

No textual amendment is presently required.

The protocol governs admission of every canonical component, record, register, and manifest.

### Specification 000006 — Work Entity Specification

No amendment is required.

Work identity remains independent of interpretation, graph representation, and engine cognition.

### Specification 000007 — Repository Cognitive Architecture

A corrected successor is required to establish:

- version-pinned cognition;
- terminology;
- canonical mappings;
- authority separation;
- Development/Production modes;
- Taxonomy/DF hierarchy;
- graph contract;
- output admission path.
